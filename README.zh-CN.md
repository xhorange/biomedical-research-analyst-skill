# biomedical-research-analyst

[![License: MIT](https://img.shields.io/badge/License-MIT-blue.svg)](LICENSE)
[![PRs Welcome](https://img.shields.io/badge/PRs-welcome-brightgreen.svg)](CONTRIBUTING.zh-CN.md)

一个用于开展严谨、可溯源的生物医学研究分析并产出学术风格幻灯片汇报的 [Claude Skill](https://www.anthropic.com/news/skills)。该 skill 将一套规范化的分析流程——数据集核实、与数据类型相匹配的统计方法、数据驱动证据与文献证据的显式区分，以及强制性的事后事实核查环节——固化为一份可复用的规范，使语言模型在不同研究课题间保持一致的分析纪律。

[English](README.md)

## 概述

大语言模型可以较为容易地产出流畅、格式规范、表面上具有权威性的生物医学分析交付物——包含基因标识符、统计量、图表以及参考文献部分。然而，此类输出的正确性并不由其呈现质量所保证。在缺乏显式核查机制的情况下，以下几类失败模式会反复出现：

1. **数据集误用。** 仅凭文件名或数据集编号即采用某数据集，未从原始记录中确认其确实对应所研究的疾病或条件。
2. **证据来源混淆。** 源自文献的论断与分析本身产出的发现以无法区分的方式呈现，缺乏可追溯的引用。
3. **通路-基因归属未经核实。** 依据一般领域知识描述某基因"属于"某生物学通路，而非依据富集分析报告的具体基因重叠列表，这可能将一个真实、显著的结果错误地归入不相关的通路。
4. **推断强度的过度表述。** 使用因果性或确证性措辞（"证明"、"证实"）替代关联性措辞，而这类措辞往往超出了横断面或观察性研究设计本身能够支撑的强度。

本 skill 是在一次实际分析中，上述每一类失败模式均被观察到、识别并加以修正之后形成的（详见下文[应用案例](#应用案例)）。与其寄望于后续分析中重新、独立地发现并修正同类问题，不如将这些修正固化为一项标准化的方法学要求。

## 目录结构

```
biomedical-research-analyst/
├── SKILL.md                          # 触发条件、分析工作流、交付前核查清单
├── references/
│   ├── dataset_verification.md       # 确认公开数据集与研究问题相匹配的核查流程
│   ├── statistical_methods.md        # 按数据类型选择统计方法、阈值预先设定、富集分析注意事项
│   ├── fact_check_checklist.md       # 强制性反查流程，并附一个真实观察到的失败案例
│   └── ppt_structure.md              # 幻灯片结构组织、引用格式、交付物的结构化校验
└── scripts/
    ├── parse_geo_series_matrix.py    # GEO series matrix 解析器（不依赖 GEOparse）
    ├── differential_expression.py    # Welch t 检验，配合 Benjamini-Hochberg FDR 校正的两组比较
    └── pptx_helpers.py               # 可复用的 python-pptx 组件（标题栏、表格、图片等比例placement）
```

`SKILL.md` 是模型在该 skill 被触发时读取的主要规范文件。`references/` 目录下的文件在工作流对应步骤按需加载，而非始终占用上下文。`scripts/` 目录下的脚本用于按需改造调用，而非作为固定不变的流水线——不同数据来源的元数据格式存在差异。

## 方法学

该 skill 规定了以下分析阶段序列：

| 阶段 | 目标 |
|---|---|
| 0 | 判断研究问题的类型（比较型、单一机制型、或危险因素综述型），以确定适用策略。 |
| 1 | 在使用任何数据集之前，依据其原始记录——标题、摘要、样本分组、检测平台——加以核实。 |
| 2 | 确定证据策略：若确有真正匹配的公开数据则据以分析；若无，则明确声明分析基于文献综述。 |
| 3 | 采用与数据类型相匹配的统计方法执行分析，阈值须在查看结果之前预先设定，所有中间结果落盘保存。 |
| 4 | 整合文献证据，附具体可核验的引用（作者、期刊、年份、PMID），并与数据驱动的发现保持显式区分。 |
| 5 | 将分析发现与已确立的生物学知识进行一致性核对，作为内部自洽性检验。 |
| 6 | 执行强制性事实核查：逐条将具体论断——通路-基因归属、汇总统计量、显著性阈值——与底层结果文件核对，并审查推断性措辞是否与研究设计相匹配。 |
| 7 | 报告局限性，包括不易察觉的混杂因素（例如：观察到的共同信号究竟是所比较条件特有的，还是某一更广泛疾病类别的通用特征）。 |
| 8 | 组装交付物，并在交付前以程序化方式校验其结构（页数、嵌入的图表数量）。 |

每一阶段背后的理由，以及促成该阶段设立的具体观察案例，均记录于 `SKILL.md` 及 `references/` 下对应文件中。

## 安装

**Claude Code：**
```bash
mkdir -p ~/.claude/skills
cp -r biomedical-research-analyst ~/.claude/skills/
```
开启新会话后该 skill 即可被调用；Claude 依据 `SKILL.md` frontmatter 中的 `description` 字段判断其适用性。

**Claude.ai：** 将该目录（或打包后的 `.skill` 文件）上传至支持自定义 skill 的工作区。

## 应用案例

该 skill 提炼自一项分析：探究**类风湿关节炎**与**动脉粥样硬化**之间是否存在共同的分子层面危险因素，所用为两个独立的 GEO 数据集（GSE55235，滑膜组织；GSE100927，动脉组织）。排除数据获取环节（该环节依赖网络条件，耗时具有不确定性），分析流程本身——解析、差异表达分析、集合交集运算、功能富集分析、组装为十二页幻灯片——完成用时约十分钟。

事实核查阶段在交付前识别出两处不准确之处：

- 一组趋化因子受体基因（CCR1、CCR2、CCR5、CCR7）在一张汇总图中被归入"白细胞跨内皮迁移"通路。经核对 Enrichr 原始输出，这组基因实为另一独立条目（"趋化因子信号通路"）的显著重叠基因，而"白细胞跨内皮迁移"这一条目的重叠基因实际为 CXCR4、ITGAL、MMP9。
- 一条关于"巨噬细胞/泡沫细胞样激活"的论断被发现只有部分成立：巨噬细胞活化确实达到统计显著性，而泡沫细胞分化这一具体条目并未达到显著性（校正后 P 值约为 0.25–0.34）。

上述两处均通过将每一条具体论断与对应结果文件逐一核对而识别——这正是 `references/fact_check_checklist.md` 中规定为强制性的流程。

## 适用范围与局限

该 skill 规范的是分析的*过程*——核实、方法学适当性、证据可追溯性，并不能替代对研究结论的专业判断。在该 skill 规范下产出的结果，除非底层数据与研究设计能够支持更强的推断，否则应被视为具有假说生成性质——而这一点正是该 skill 局限性阶段被设计为明确指出的内容。
