# 贡献指南

[English](CONTRIBUTING.md)

与该 skill 自身的设计原则一致，其演进应基于具体、已记录的真实案例，而非未经验证的一般性经验法则。以下几类贡献尤为有价值：

## 报告观察到的失败模式

如果你在使用该 skill 的过程中，发现某个具体的错误论断绕过了强制性事实核查环节最终出现在交付物中，或者发现某一类研究问题被处理得不理想，请提交 issue 并说明：

1. 交给模型的具体研究问题或任务；
2. 具体是哪个论断或输出结果有误，以及错在哪里；
3. 工作流中的哪个环节（`SKILL.md` 里的哪个阶段，或 `references/` 下的哪份文件）本应能够拦截这个错误，但实际没有拦截住。

具体、可复现的失败案例是该 skill 得以改进的主要途径；没有具体案例支撑的泛泛建议依然欢迎讨论，但相对更难落地为具体改动。

## 提出改动

欢迎针对以下方向提交 Pull Request：

- 为 `references/` 补充目前尚未覆盖的数据类型、统计方法或数据仓库的指南（例如蛋白组学、单细胞数据、GEO 之外的其他数据库）；
- 改进或扩展 `scripts/` 下的脚本——前提是改动后它们依然保持"可按需改造的组件"这一定位，而不是变成一套僵化的、试图适配所有情况的固定流水线；
- 打磨 `SKILL.md` 本身的措辞，尤其是把某条要求背后"为什么这么要求"讲得更清楚，而不是留一条没有解释理由的规则。

提交 Pull Request 时，请说明促成这次改动的具体场景——该 skill 本身要求生物医学论断要有可追溯的依据，这个标准同样适用于这里：有具体来龙去脉的改动，优先于没有实例支撑的泛泛而谈的"改进"。

## 范围界定

该 skill 规范的是一套分析工作流，而不是某个具体的生物医学领域。能让工作流变得更严谨、更通用、或者解释得更清楚的贡献都在范围内。而那些把假设写死在某一个具体疾病领域或某一个具体数据集上的贡献，除非被抽象、泛化成一份适用范围更广的 `references/` 指南，否则大概率不在此范围内。
